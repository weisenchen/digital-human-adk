# Talk Show Mode — Plan

> **Status**: Planning only. Read Script is preserved unchanged.
> New feature: "Talk Show" — a structured interview mode where the Digital Human acts as a talk show host.

## Goal

Add a **Talk Show** mode alongside the existing Read Script. In this mode:

1. **User prepares**: topic, guest name/info, background materials, interview questions
2. **Host (Digital Human)**: uses the prepared content to conduct the interview
3. **User = Guest**: answers as the interviewee via voice or text
4. **AI responses grounded**: the host's questions and reactions stay within the prepared material
5. **Avatar animates**: mouth moves during speech, same as voice chat

The existing Read Script (slide presentation) remains untouched.

## How It Works — User Flow

### 1. Setup Stage (Sidebar or Popup)

User opens a "New Talk Show" option (in sidebar, below Read Script):

Form fields:
- **Host name** (defaults to current Digital Human character name)
- **Guest name** (who is being interviewed)
- **Topic / Theme** (e.g., "The Future of AI in Healthcare")
- **Background materials** (textarea): paste articles, notes, context, key points
- **Interview questions / outline** (textarea, optional): list of questions to guide the show
- **Duration / style**: casual interview, deep-dive, panel-style

### 2. Start → Talk Show Mode

The page switches to a **Talk Show layout**:
- **Left side**: Digital Human avatar (same size as normal chat)
- **Right side**: 
  - Top: show title, guest name, current topic
  - Middle: Chat bubbles (host questions + guest answers)
  - Bottom: Input bar (user types/speaks as guest) + "End Show" button

### 3. During the Show

- **Host speaks first** (AI generates opening based on background, introduces guest)
- **User responds** via text or voice (as the guest)
- **Host follows up** based on:
  - Prepared background materials (primary source)
  - Prepared questions list (if provided)
  - The guest's actual responses (real-time adaptation)
- **Avatar's mouth moves** during host speech
- **Slides / visuals** can optionally appear (future enhancement)

### 4. End Show

User clicks "End Show" → summary/log saves, returns to normal chat.

### Backend: Talk Show Agent

A new endpoint `POST /api/talk-show/ask` that:

1. Receives:
   - `session_id`
   - `background` (user's prepared materials)
   - `questions` (optional interview outline)
   - `topic`
   - `guest_name`
   - `host_name`
   - `message` (latest guest response)
   - `history` (conversation so far)
2. Constructs a system prompt that:
   - Instructs the AI to act as the host character
   - Grounds responses in the background materials
   - Follows the prepared questions outline
   - Maintains interview structure
3. Returns the host's response (text, with SSE streaming for real-time feel)

### Frontend: Talk Show Components

**TalkShowSetup** (popup/modal):
- Form for topic, guest, background, questions
- "Start Show" button

**TalkShowMode** (main view):
- Split layout: avatar + interview chat
- Shows host speech with mouth animation
- Input for guest responses
- "End Show" button

## Important Distinctions

| Aspect | Read Script (existing) | Talk Show (new) |
|---|---|---|
| Format | Pre-written script → slides | Live interview / conversation |
| Avatar | Hidden behind overlay | Visible, animated |
| User role | Passive viewer | Active participant (guest) |
| Content | Linear slides | Structured conversation |
| AI role | Slide generator | Talk show host |
| Interaction | None (click to advance) | Real-time conversation |
